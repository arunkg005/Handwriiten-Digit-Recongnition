Understanding the Core

A simple look at how the app turns handwritten strokes into clean, structured predictions with fast feedback and region-aware analysis.

About the Project

This web app demonstrates real-time handwritten digit recognition through an interactive canvas. It helps users see how strokes are interpreted, separated, and mapped into stable predictions.

Our Model

The predictor is powered by a local MNIST-trained MLPClassifier(hidden_layer_sizes=(256, 128)). A binary rejector then filters uncertain marks and non-digit strokes before the UI commits a final answer, while the pipeline still normalizes strokes, detects separate digit regions, and scores each active region with confidence values.

Pipeline

01

Capture Stroke

The browser records what you draw, tracks the latest selection hint, and keeps the active stroke in sync with the live canvas.

02

Normalize

Converts the drawing into a 28×28 MNIST frame, normalizes pixels, and centers the digit to match training data distribution.

03

Segment

Foreground regions are split so the app understands one digit at a time, even when the canvas has multiple marks.

04

Classify

A local MLPClassifier(256, 128) scores every candidate region and produces the full 10-class probability breakdown.

05

Gate & Reject

A binary rejector cross-checks classifier confidence, bounding-box geometry, and stroke density to block uncertain or non-digit input.
